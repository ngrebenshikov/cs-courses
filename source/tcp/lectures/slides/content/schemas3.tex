\begin{slide}
	\textbf{Рекурсивное программирование} - метод формализации вычислимой функции основанный на связи искомого значение функции для заданных аргументов с известными значениями той же функции при некоторых других аргументах.
	
	Что это за функции?
	
	$\displaystyle f(x) = \begin{cases} 1, & x = 1 \\ x \cdot f(x-1), & x > 1 \end{cases}$
	
	$\displaystyle f(x) = \begin{cases} 1, & x = 1 \vee x = 2 \\ f(x-1) + f(x-2), & x > 2 \end{cases}$
\end{slide}

% ----------------------------------------------------------------
\begin{slide}
	\textbf{Базис рекурсивной схемы} есть расширение базиса ССП
	
	В множество специальных символов добавлено \newline $\{if, then, else, (, ),\ , , =\}$.
	
	В множество функциональных символов добавлены определяемые функциональные символы. Например, $F^{(1)}, G^{(2)}$.
\end{slide}

% ----------------------------------------------------------------
\begin{slide}
	\textbf{Термы рекурсивной схемы}
	
	Базовые термы - $f(x, g(x, y)); h(h(a))$
	
	Простые термы - $f(F(x), g(x, F(y))); H(H(a))$
	
	Вызовы - $F(x); H(H(a))$

	Условный терм - $ \textbf{if} \ p(x, y) \ \textbf{then} \ h(h(a)) \  \textbf{else} \ F(x)$
\end{slide}

% ----------------------------------------------------------------
\begin{slide}
	\textbf{Рекурсивное уравнение} (определение функции) $F$
	
	$F^{(n)}(x_1,x_2,\ldots, x_n) = t(x_1,x_2,\ldots, x_n)$, 
	
	где $t(x_1,x_2,\ldots, x_n)$ - 
	терм, содержащий переменные, называемые формальными параметрами функции $F$.
\end{slide}

% ----------------------------------------------------------------
\begin{slide}
	\textbf{Рекурсивная схема} - пара $(t,M)$
	
	где $t$ - терм, называемый главным термом схемы (или ее входом). $M$ - такое множество рекурсивных уравнений, что все определяемые функциональные символы в левых частях уравнений различны и всякий определяемый символ, встречающийся в правой части некоторого уравнения или в главном терме схемы, входит в левую часть некоторого уравнения.
	
\end{slide}

% ----------------------------------------------------------------
\begin{slide}
	\textbf{Примеры рекурсивных схем}
	
$R_{S1}: \ F(x); \ F(x) = if \ p(x) \ then \ a \ else \ g(x, F(h(x))).$

$R_{S2}: \ A(b, c); \ A(x, y) = if \  p(x) \ then \ f(x) \ else \ B(x, y);$

$B(x, y) = if \  p(y) \ then \ A(g(x), a) \ else \ C(x, y);$

$C(x, y) = A(g(x), A(x, g(y))).$

$R_{S3}: F(x); F(x) = if \  p(x) \ then \ x \ else \ f(F(g(x)), F(h(x))).$
\end{slide}

\begin{slide}
	\textbf{Счетчиковые схемы}

	\textbf{Счетчик} — интерпретированная переменная, у которой областью значений является множество $N$; начальное значение счетчика равно 0.

	Интерпретированные операторы имеют следующий вид:
	
\begin{itemize}
	\item c := c + 1 — оператор прибавления единицы;
	\item c:= c - 1  — оператор вычитания единицы;
	\item c = 0 — условный оператор проверки равенства счетчика нулю.
\end{itemize}
\end{slide}

% ----------------------------------------------------------------
\begin{slide}
	\textbf{Магазинные (стековые) схемы}

\textbf{Магазин (стек)} — неинтерпретированная переменная сложной структуры. В процессе выполнения интерпретированной схемы состояние магазина — это конечный набор элементов $(d_1,d_2,\ldots,d_n)$ из области интерпретации, где $d_n$ — верхушка магазина.

Интерпретированные операторы имеют следующий вид:
\begin{itemize}
	\item М := x — запись в магазин;
	\item х := М — выборка из магазина;
	\item М = $\oslash$ — условный оператор проверки пустоты магазина
\end{itemize}
\end{slide}

% ----------------------------------------------------------------
\begin{slide}
	\textbf{Схемы с массивами}

	\textbf{Массив} — неинтерпретированная переменная сложной структуры. При выполнении интерпретированной схемы состояние массива — бесконечная последовательность $(d_1,d_2,\ldots,d_i,\ldots)$ элементов из области интерпретациии.

Интерпретированные операторы имеют следующий вид:
\begin{itemize}
	\item А[c]:= x — запись в массив;
	\item х:= А[c] — выборка из массива
\end{itemize}

где А — массив, [c] — целое число, равное текущему значению счетчика с.	
\end{slide}

\begin{slide}
	\textbf{Структурированные схемы} - ССП с добавлением операторов условия и цикла.
	
	Условный оператор - $if \ p \ then \ s_1 \ else \ s_0 end$, где $p$ — логическое выражение, $s_1, s_0$ — последовательности (может быть, пустые) операторов, среди которых нет ни начального, ни заключительного.
	
	Оператор цикла - $while \ p \ do \ s \ end$ или $until \ p \ do \ s end$
\end{slide}
